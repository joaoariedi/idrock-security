BEDROCK - SISTEMA DE ANÁLISE DE REPUTAÇÃO DE ACESSO PARA DETERMINAÇÃO DE RISCO DE FRAUDE


Relatório de apresentação de proposta de solução de análise de reputação de acesso para determinação de risco de fraude no login de usuários da empresa NexShop.
























1. Objetivo
O objetivo do projeto a coleta de informações das necessidades da empresa proponente (cliente), compreensão dos requisitos chave, pesquisa de soluções às necessidades apresentadas, elaboração de um SDK (Software Development Kit) que vise construir uma solução para detectar tentativas de fraude no momento do login, checkout ou em ações sensíveis, com o mínimo de fricção para o usuário final.
2. Introdução
Ao avaliar as necessidades apresentadas pela empresa, o grupo definiu que a solução que se adequa é uma biblioteca em Javascript que analisa e responde uma classificação de risco com base nas informações coletadas no frontend do cliente por meio de um módulo integrado e envia via HTTP para uma rota definida no backend, configurando então um kit de desenvolvimento de software.
A estratégia de desenvolvimento adotada pela equipe será a implementação das verificações menos invasivas, que geram menor fricção, havendo ocorrência de alerta de segurança o sistema progride para verificações mais complexas e invasivas.
3. Funcionalidades previstas para o mínimo produto viável


   1. Reputação de Endereço IP[a]
O uso de ferramentas de análise de reputação de IP é uma prática essencial tanto na segurança cibernética quanto na comunicação digital. O principal objetivo é avaliar a confiabilidade de um endereço de internet com base em seu comportamento histórico e atual.


A ferramenta que será utilizada será o ProxyCheck.io (https://proxycheck.io).
________________


   2. Tipo de conexão
Técnicas de evasão conhecidas se utilizam de conexões indiretas para mascarar acessos, para evitar essa técnica será implementada uma avaliação do tipo de conexão do usuário, analisando se há anomalia histórica que pode configurar risco de fraude, observando se o usuário está usando:
* VPN
* Proxy
* TOR


A ferramenta que será utilizada será o ProxyCheck.io (https://proxycheck.io).


   3. Localização Geográfica[b]
Para avaliar possíveis acessos não autorizados por um provedores de Internet diferente do habitual ou um caso de viagem impossível, uma avaliação de Autonomous System Numbers (ASNs) pode ser realizada, as coordenadas de GeoIP, assim como a identificação da distância do acesso em relação ao endereço cadastrado pelo usuário para Geo Fence. 


A ferramenta que será utilizada será o ProxyCheck (https://proxycheck.io).


   4. Histórico de dispositivo[c]
Rastreio do dispositivo por fingerprint do web browser.


A ferramenta que será utilizada será eleita desta lista:
* FingerprintJS (https://github.com/fingerprintjs/fingerprintjs)
* ThumbmarkJS (https://github.com/thumbmarkjs/thumbmarkjs)


   5. Comportamento temporal
Análise comportamental do usuário baseada no histórico de horários, dias da semana e dispositivos usados (usando histórico de dispositivo) para acessar a aplicação, visando identificar anomalias de comportamentos no uso.


Será alimentado um histórico de acesso em banco de dados para cruzamento de informações.




   6. Computador Real[d]
Analisar o hardware mínimo de um computador real visando excluir ferramentas que não são um browser real em um computador real.


Serão verificados por Javascript do web browser a quantidade de memória e de cpus para determinar se é um computador real:


  



   7. Browser Real[e]
Avaliação passiva via HTTP User-Agent do lado do servidor para determinar se é um Bot ou ferramenta automatizada.


Será utilizada esta lista de validação:
https://github.com/mitchellkrogza/nginx-ultimate-bad-bot-blocker/tree/master/_generator_lists


Para determinação de Browser Headless, visando identificar tentativas de acesso via scripts e automações que usam uma engine de browser real mas sem interface, será utilizado está biblioteca:
https://github.com/fingerprintjs/BotD
________________


   8. Captcha Invisível com Prova de Trabalho[f]
* Resistente a replay
* Torna a força bruta lenta e pesada
* Fácil para humanos e difícil para bots


A ferramenta que será utilizada será o CapJS (https://capjs.js.org)

As funcionalidades descritas abaixo podem ser incorporadas conforme viabilidade e prioridade do produto.
